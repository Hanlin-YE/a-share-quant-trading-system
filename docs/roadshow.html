01 · 背景与问题

散户做 A 股量化，卡在哪？

数据散、工具碎、回测过拟合、策略无治理、风控靠感觉——大多数个人量化项目停在"能跑一个指标"，走不到"可复盘的闭环"。

  • 数据层不稳定：行情源经常断流、复权不一致、缓存没健康检查，回测在脏数据上跑。
  • 研究即弃：分析完一只票，结论留在聊天记录里，没有观察池、thesis、卖出条件。
  • 策略无版本：参数手调、没有样本外验证、没有 walk-forward，回测好看实盘崩。
  • 执行无纪律：没有日内流程、没有风控阈值、没有失败桶反馈，亏了不知道为什么。
  • LLM 用法粗糙：把整段行情/新闻塞 prompt，token 烧光还效果差。
02 · 方案概述

一个项目，五个子系统，一条闭环

不是"一个脚本"，而是按职责拆分的五模块系统：研究、短线、生命周期、账本、工具各自独立，通过 CSV/JSON 契约衔接。

stock-analyzer

量化主引擎。四引擎框架 + ML 概率集成 + 多源行情适配 + 组合回测 + 策略版本注册表。

Shenzhen intern trail

A 股短线题材筛选管道。新闻/热搜 → DeepSeek 题材提取 → 四层规则筛选 → 龙二/龙三买点计划。

stock-research-lab

研究生命周期。观察池、买入 thesis、卖出条件、复查日期、研究健康度。

trading-journal

纸面账户与交易日志。唯一组合总表 + 五阶段流水线记录 + 复盘文档 + 策略反馈。

tools

账本工具链。Excel 工作簿同步、历史回填、组合账本重建。

闭环契约

CSV/JSON 作为模块间唯一事实源，不共享数据库、不共享内存状态，任何一环可独立替换。

03 · 系统架构

数据从左到右流过五个阶段

📡
数据层
推送缓存 / Tushare / 腾讯 / 东财 / Stooq，复权与缓存健康门禁
🧮
特征层
pandas 因子工程：均线、RSI、KDJ、MACD、ATR、Z-score、回撤
🤖
模型 / 规则层
ML 集成概率 + 聚类 + 马尔可夫 + 贝叶斯，叠加四引擎规则
⚖️
风控 / 回测
最大回撤、波动率、仓位上限、止损，回测显式计入佣金印花税滑点
📒
执行 / 账本
五阶段流水线 → 纸面账户 → 组合总表 → 策略治理反馈

核心设计原则：风控结论优先于买入信号；研究回测与纸面账户口径分离；所有自动决策 dry-run 默认，--execute 才落账。

04 · 数据层

多源行情适配 + 缓存健康门禁

A 股数据是项目最容易翻车的一层。系统用"专业优先 + 多源兜底 + 健康门禁"三段式保证回测不在脏数据上跑。

分级数据源

premium推送缓存优先
tushareTushare Pro 主动拉取
tencent腾讯前复权日线
eastmoney东方财富兜底
stooq实验源

健康门禁机制

--health-check 批量校验哨兵股票（600519 / 000001 / 300750）历史与实时兜底。任一失败即阻断完整分析与自动化复盘，避免空数据报告。

缓存与导入

支持本地 CSV 导入（中英文列名兼容），--cache-health 检查新鲜度，--cache-max-age-hours 控制可回测子集。

字段规范化

自动识别沪深市场、前复权对齐、交易日对齐、缺失值检查、symbol 规范化（removeprefix("^") 防 ^^GSPC 别名问题）。

05 · 量化引擎 stock-analyzer

四引擎框架 + ML 集成 + 仓位 Profile

1 · 题材轮动引擎

从涨停归因表统计涨停数 / 题材强度 / 资金净额 / 封单强弱，取前三大核心题材，只在题材内筛未涨停候选。

2 · 量价技术引擎

RSI、KDJ、MACD、均线、ATR、波动率、回撤，单票诊断 + 批量扫描留痕。

3 · 投研证据链引擎

候选股必须说明与题材龙头的产业链 / 业务 / 事件关联，叠加行业龙头、情绪龙、机构持仓、研报覆盖线索。

4 · 回测风控引擎

所有看多信号被交易成本、样本外验证、最大回撤、波动率、仓位上限、止损条件约束。风控优先于买入。

监督学习 决策树 / 随机森林 / ExtraTrees / 逻辑回归 / SVM / KNN / MLP
验证 TimeSeriesSplit + gap 防泄漏
状态 KMeans 聚类 + 马尔可夫转移 + Beta-Binomial 胜率后验
Profile llmquant-long-biased / quant-paper / serenity-chokepoint / local
06 · 题材短线管道 Shenzhen intern trail

新闻 → DeepSeek → 四层规则 → 买点计划

独立于主引擎的短线题材筛选原型，每 30 分钟扫描一次，输出研究用候选，不自动下单。

LAYER 1
新闻/热搜扫描
金十 / Wind / 百度热搜 / Google Trends / 官媒，DeepSeek 提取热点题材与个股池
→
LAYER 2
技术与风险过滤
主力净流入 / 换手有效，剔除 ST / 停牌 / 重大诉讼 / 业绩预警 / 监管调查
→
LAYER 3
量与大单过滤
爆量或量能 4>11>117 多头排列，大单占比高，MA5>MA10>MA20，连跌 4 天否决
→
LAYER 4
买点计划
同板块已有涨停锚点，优先龙二；龙二快速封板切龙三；A/B 类突破或 7-8% 打板挂单

运行状态用 PASS NO_SIGNAL BLOCKED 三态明确区分"有候选 / 无候选 / 数据源失败"，不会把失败伪装成空结果。

07 · 策略版本与回测

样本外验证 + Walk-forward，过拟合兜底

策略不是手调参数，而是注册表 + 训练/验证/测试三段切分 + 滚动样本外复核，晋级有门槛。

策略注册表

strategies.json

score-standard-v1 / score-conservative-v1 / score-active-v1，每个策略有 entry/exit 分数、仓位上限、最小样本量。

三段切分

train / val / test

候选只按验证集排序，测试集只用于样本外复核，不用测试集挑策略。

滚动复核

walk-forward

多窗口必须有交易且平均收益为正，减少"单一切分刚好运气好"的过拟合。

晋级门禁 promotion_gate

即使候选通过门槛，也只能进入纸面交易观察，不允许自动真实下单。failed_rules 和 next_search.actions 告诉下一轮自动调参应该先扩缓存池、放宽/收紧阈值，还是进入纸面观察。

08 · 日内五阶段流水线

从盘前到收盘，每个阶段可独立审计

08:30
preopen 盘前
缓存门禁 + 可选策略优化/晋级，写入 STRATEGY_FEEDBACK
→
09:30
open 开盘
按 active paper strategy 生成 BUY/SELL/HOLD 决策包
→
12:00
midday 午间
复核上午结果，生成下午计划
→
13:00
afternoon 午后
按午间计划/active strategy 生成执行决策
→
15:10
close 盘后
收盘决策包，--execute 才结算纸面账本并写运营日报

默认 dry-run

不写订单、不结算、不真实下单。只有显式 --execute 才写入纸面账本；只有 preopen --promote 才允许策略晋级。

可复盘

每个阶段写入 pipeline-runs/<日期>/<阶段>.json，pipeline_review.py 输出失败桶：data_cache_unhealthy / open_failed / pipeline_ok。

10 · 交易日志与策略治理

唯一组合总表 + 反馈闭环

所有交易事实收拢到 portfolio_ledger.csv 一张表，按 row_type 区分：DAY_SUMMARY / POSITION / TRADE / STRATEGY_FEEDBACK。Excel 工作簿是审计视图，CSV 是机器事实源。

流程检查账本

workflow_checkpoints.py 把盘前 / 开盘 / 10:30 / 午间 / 14:00 / 收盘 / 盘后审计写入 workflow-checkpoints.csv，错过时段事后 audit 标 overdue，不让"待验证"长期沉没。

策略治理器

strategy_governance.py 把反馈桶和账户风险状态转成下一轮权限：是否允许研究回测 / 参数搜索 / 策略晋级 / 新增仓位。STOP 或单日亏损 >1% 的策略先 probation 降权，连续失败 3 次才淘汰。

显示规范

方向统一中文（买入/卖出/持有/观察/研究），判断状态固定枚举（已成交/持仓中/已完成/待处理/失败/已豁免/逾期未留证）。回测行不得占用正式账户权益列。

账本连续性

第二天期初权益必须等于前一日 DAY_SUMMARY 期末权益；不一致先修账，不继续提交审计。BACKTEST 与组合汇总口径分离，不可相加。

14 · 风险提示

研究工具，不是投资建议

本系统输出仅供量化研究和辅助决策，不构成投资建议，不会自动下单。A 股存在涨跌停、停牌、流动性、政策冲击和数据源稳定性问题。账户为纸面训练账户（paper account），所有"盈亏"均为模拟。进入实盘前必须补齐全市场回测、交易成本建模、样本外验证、组合级风控、券商接口、撤单逻辑和人工复核。

所有自动决策默认 dry-run，--execute 才写入纸面账本；preopen --promote 才允许策略晋级；即使策略通过晋级门禁，也只能进入纸面交易观察，不允许自动真实下单。
  • 本仓库不含真实交易凭证、不含真实资金、不含未公开数据。
  • DeepSeek API key 等敏感配置已通过 .env + .gitignore 排除，不会进入版本控制。
  • 行情数据来源为公开接口（腾讯 / 东方财富 / Tushare / Stooq），使用权遵循各源条款。

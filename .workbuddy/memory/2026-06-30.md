# 2026-06-30 工作日志

## 调研任务：LangAlpha token 节省机制 + 本项目开源构成

### 1. LangAlpha 仓库调研（github.com/ginlix-ai/LangAlpha）
- 核心机制为 **PTC (Programmatic Tool Calling)**：LLM 不直接调 MCP 工具，而是写 Python 代码经 `execute_code` 在 Daytona 沙箱中执行；沙箱内 `ToolFunctionGenerator`/`generate_mcp_client_code()` 生成 MCP wrapper，调 FMP/GinlixData/SEC 等金融数据工具，用 pandas/numpy 处理，只把精简结果回传 LLM。
- token 节省的多层设计：
  - `SNIPPET_MAX_CHARS` 截断 + `provenance_result_bodies` 完整结果存 DB/对象存储，不进 SSE 流（"SSE stream stays lean"）
  - `downsampleBars` K 线降采样 + bar cap
  - `ChartSelectionContext` 结构化上下文替代原始 OHLCV
  - memo store 预计算缓存
  - compaction / DeltaChannel 上下文压缩（每线程准入门控）
  - `conversation_usages` 持久化 token 计量
- 关键目录：`src/data_client/{fmp,ginlix_data}`、`src/ptc_agent`、`src/server`、`src/config`。

### 2. 当前项目开源构成盘点
项目极度轻量，**不依赖任何大型开源框架**，与 LangAlpha（基于 LangGraph/deepagents/FastAPI/React）形成鲜明对比。
- Python 实际使用第三方库：pandas、numpy、requests、openpyxl
- requirements.txt 声明但代码未直接 import：pandas-datareader、scikit-learn、scipy
- LLM 集成：DeepSeek API 用 urllib 原生 HTTP 调用（未用 openai SDK）
- 数据源适配器：东方财富、新闻、DeepSeek 均用 stdlib urllib
- 前端 stock-analyzer/web：纯 vanilla JS，无框架、无构建工具、无 package.json
- Node 工具 inspect_portfolio_workbook.mjs：引用 `@oai/artifact-tool`（宿主环境内部工具，非公开开源包，无 node_modules）
- subprojects：stock-analyzer、Shenzhen intern trail、stock-research-lab、trading-journal、tools 均为自包含 Python 脚本
